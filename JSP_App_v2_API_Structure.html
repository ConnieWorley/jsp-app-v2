API Structure

Phase 2 — Architecture  ·  Client-side · React → Supabase + Claude API

Architecture Decisions Locked
◆All API calls are client-side (React). No server middleware or Edge Functions in v2.
◆JD import = URL field (stored for reference only) + JD paste. No fetch or scrape attempt.
◆Claude API generates output; Supabase stores the result. Both called in sequence where needed.
◆Single-option fields are PostgreSQL enums. Multi-select fields are text[] arrays.

Core AI Sequence — JD Import to Scorecard
1
User pastes JD text (and optionally the URL for reference)
No fetch attempted. Clean paste UX.
2
React saves job record to Supabase (jd_text, jd_url, status=draft)
Supabase insert → jobs table
3
React calls Claude API with JD + user profile (role target, resume label, ideal company criteria) as context
Single prompt returns: three scores + rationale + skills gaps + gap severity + training recs
4
React parses Claude response (structured JSON output)
Scores → jobs table · gaps → skills_gaps table · training recs → skills_gaps.training_rec
5
React calls Claude API again with JD + storybank to rank story matches
Returns ranked list → display only, no separate storage needed
6
Scorecard, gap analysis, and story matches render for user review
User can override status, edit items, or proceed to application stage
